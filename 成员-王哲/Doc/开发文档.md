## 四、任务1：标准时空数据变换（滑动平均趋势提取）

### 4.1 目标
将抖动的原始SST时间序列平滑成趋势线，作为预测模型的前置输入。对应 Tamara 抽象层数据抽象。

### 4.2 实现要求
- 文件：`code/task1_smoothing.py`
- 按 `region` 分组，每个海域独立按 `time` 排序后做滑动平均
- 同时实现两种平滑（供对比）：
  - 简单滑动平均：`df['sst'].rolling(window=W, min_periods=1).mean()`
  - 指数加权平均：`df['sst'].ewm(span=W).mean()`
- 窗口 W 默认 10，作为可调参数（前端滑块会复用此参数）
- 全部使用 pandas 自带函数，不手写公式

### 4.3 输出
`output/smoothed.json`：每个海域的 `[{time, sst_raw, sst_ma, sst_ewm}]`，供前端时序图渲染原始点+两条平滑线。

## 五、任务2：轻量AI机器学习模型（多变量SST预测）

### 5.1 目标
不只用时间预测温度，而用**多维度耦合关系**预测SST。对应 Tamara 任务层，覆盖 Analyze（高级）/Search（中级）/Query（低级）三级子任务。

### 5.2 三级分析任务定义（实验强制要求，必须写清）
- **Query（低级）**：查询某海域某时间的SST实测值与平滑值
- **Search（中级）**：搜索满足多维阈值条件的记录（如 CO₂>420 且 风速<10 的高温样本）
- **Analyze（高级）**：分析16维耦合关系，用多变量模型预测SST，并量化各维度对预测的贡献

### 5.3 特征工程
预测SST使用的输入特征（基于物理相关性筛选）：
```
latitude, co2, wind_speed, salinity, humidity, pressure, current_speed
```
- 类别字段 `region` 做 one-hot 编码
- 加入时间特征：从 `time` 提取 `month`（捕捉季节性）
- 所有特征用 `StandardScaler` 标准化（沿用任务1的归一化思想）

### 5.4 模型
- 文件：`code/task2_predict.py`
- 主模型：随机森林回归 `RandomForestRegressor`（能输出特征重要性，适合耦合分析）
- 对比模型：线性回归 `LinearRegression`（提供可解释的系数）
- 训练/测试划分 8:2，输出 R²、MAE、RMSE 评估指标
- **必须实现的接口**（为未来接昇腾预留）：
```python
def predict(features: pd.DataFrame) -> np.ndarray:
    """统一预测入口。当前本地 sklearn 实现；
    未来接 ModelArts 时只替换函数体，调用方不变。"""
    ...
```

### 5.5 输出
- `output/prediction.json`：测试集真实值 vs 预测值、各海域预测误差
- `output/feature_importance.json`：各维度对SST预测的重要性/贡献
- `output/forecast.json`：基于趋势的未来延伸预测 + 置信区间

## 六、降维耦合分析（创新可视化的数据基础）

### 6.1 目标
16维太多无法直接观察，用 PCA 压缩到3维，在3D空间展示海域在多维耦合空间中的分布差异。

### 6.2 实现
- 文件：`code/coupling_pca.py`
- 对标准化后的16维（去掉id/time/region/wind_dir）做 `PCA(n_components=3)`
- 输出每条记录的三个主成分坐标 + 原始region/co2/sst
- 同时计算16维两两 Pearson 相关系数矩阵（特征耦合热力图用）

### 6.3 输出
- `output/pca_points.json`：`[{pc1, pc2, pc3, region, co2, sst}]`
- `output/correlation_matrix.json`：16×16 相关系数矩阵
- `output/pca_explained.json`：三个主成分的方差解释率

## 七、可视化界面（核心产出）

实验要求：1主视图 + 2-3辅助视图（含创新图），严格落实可视化三部曲（筛选→映射→交互）。

### 7.1 主视图（创新图表）——三维耦合星云图

**标注【本组创新图表】**

- 技术：**Plotly.js** 3D scatter
- X/Y/Z = PCA三个主成分（综合了温度/CO₂/风速等多维信息）
- 颜色编码 = 海域（Okabe-Ito色盲友好色板，沿用实验2规范）
- 点大小编码 = CO₂浓度
- **创新点**：
  - 在三维空间中展示"海域在多维耦合空间的聚类分布"，普通折线/散点无法表达
  - 框选一批点 → 下方辅助视图联动显示这批点的预测温度分布
  - 可旋转、缩放、悬浮看详情
- **视觉隐喻**：每个海域像一团"星云"，聚集程度反映该海域多维特征的内聚性
- **解决的分析问题**：常规图表无法同时表达"16维耦合关系+海域差异+温度分布"，星云图用降维把高维耦合压进可旋转的3D空间

### 7.2 辅助视图1——16维特征耦合热力图
- 技术：D3.js
- 16×16 相关系数矩阵，颜色深浅=耦合强度（顺序色阶，蓝白红发散）
- 点击某个格子 → 高亮对应两个维度，主视图按这两维重新着色
- 回答"哪些维度互相影响最强"

### 7.3 辅助视图2——多维时序预测图
- 技术：D3.js
- 原始SST（半透明散点）+ 滑动平均趋势线（实线）+ 未来预测（虚线）+ 置信区间（渐变色带）
- 顶部滑块：拖动调节滑动窗口W，实时重算平滑曲线（交互体现可视化三部曲）
- 海域切换下拉框
- 回答"温度过去怎么变、现在什么趋势、未来会怎样"

### 7.4 辅助视图3——预测贡献瀑布图
- 技术：D3.js
- 选中一个预测点，用瀑布图拆解：基准值 → +纬度贡献 → +CO₂贡献 → +风速贡献 → … → 最终预测值
- 颜色：正贡献暖色、负贡献冷色
- 回答"这个温度预测值，是哪些维度怎么影响出来的"

### 7.5 多视图联动（实验强制要求）
- 主视图星云图框选 → 辅助视图2/3更新为选中样本
- 热力图点击维度 → 主视图重新着色
- 时序图海域切换 → 全局过滤该海域
- 数据同步逻辑：统一一份前端数据状态，各视图订阅同一筛选条件
