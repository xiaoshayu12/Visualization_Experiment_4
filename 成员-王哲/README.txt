================================================================================
 海洋-大气多维耦合分析平台  ·  成员 A 王哲
 实验4：算法集成与 AI 增强分析（统一系统主页）
================================================================================

本系统是「全球海洋大气耦合时空可视分析系统」的主页子系统。围绕同一份约 3600 条
海洋-大气观测数据（6 海域 × 2020–2022），用一条本地算法管线把高维耦合关系、
温度趋势与多变量预测全部预计算成静态数据，再由前端五个联动视图呈现。整套系统
本地优先、无云端依赖，既能由统一 Flask 后端在 "/" 路由提供，也能脱机单独查看。


一、五大视图（顶部导航单屏切换，不滚动）
--------------------------------------------------------------------------------
  1) 总览          —— 海域碳-温状态总览：数据规模 KPI、各海域 SST 均值/极值、
                       特征重要性、PCA 方差解释率、两模型精度对比。分析的入口。
  2) 耦合星云 ★    —— 主视图 / 本组创新图表：把 12 维海洋-大气特征经 PCA 压进
                       可旋转的三维空间，每个点是一条观测，按海域着色、按 CO₂
                       定大小。可框选一批样本，联动下方「演变」「归因」视图。
  3) 维度互动强度  —— 13 维两两 Pearson 相关矩阵（蓝-白-红发散色）。点击任一格，
                       星云图即按该维度连续重新着色，验证其空间分布。
  4) 海温演变与展望—— 某海域 SST 的原始散点 + 滑动平均趋势（窗口 W 滑块实时重算）
                       + 随机森林未来半年预测虚线 + 95% 置信带。
  5) 单点预测归因  —— 选定一个测试样本，用线性模型把预测温度逐维拆解成瀑布：
                       从基准值出发，每个维度把温度往上(暖)/往下(冷)推了多少。

  三级分析任务（Tamara 任务层）：
    · Query（查询） —— 左侧信息面板「数据点查询」：查某海域某日的实测/平滑 SST。
    · Search（搜索）—— 顶栏「多维筛选」：按 CO₂/风速/盐度等多维阈值检索记录。
    · Analyze（分析）—— 后三个视图：耦合、预测与贡献归因。


二、技术栈
--------------------------------------------------------------------------------
  算法：Python 3.11 + pandas + numpy + scikit-learn（conda 环境名 ocean-vis）
  前端：原生 JS + D3.js v7（2D 图表）+ Plotly（3D 星云），均本地引入，无 CDN
  数据流：算法管线 → output/*.json → 打包 data-bundle.js → 前端 window.OCEAN_DATA


三、运行方式
--------------------------------------------------------------------------------
  方式①（推荐，随统一系统启动）：
      在项目根目录执行  python backend/app.py  ，浏览器访问 http://localhost:5000/
      统一后端把本页作为主页提供，并桥接 css/ js/ lib/ 相对资源，
      顶部显示四位成员的统一导航栏。

  方式②（脱机单独查看）：
      直接双击 frontend/index.html 即可看五视图（算法产物已打包进 data-bundle.js，
      绕过 file:// 对本地 JSON 的读取限制，无需任何服务）。
      注：脱机模式下顶部「四页跨成员导航栏」与全局样式不加载（它们用绝对路径
      /static/ 指向统一后端），但本系统五视图功能完整不受影响。
      3D 星云依赖 WebGL，请用 Chrome / Edge / Safari 等现代浏览器。
      地址栏 #nebula / #coupling / #forecast / #waterfall 可直达对应视图。


四、重跑算法管线（修改算法后才需要）
--------------------------------------------------------------------------------
  首次准备虚拟环境（二选一）：
      conda env create -f environment.yml
      conda create -y -n ocean-vis python=3.11 pandas numpy scikit-learn

  执行：
      conda activate ocean-vis
      cd code
      python run.py          # 串联全部算法，生成 output/*.json 并自动打包 data-bundle.js

  run.py 会依次跑 preprocess → task1_smoothing → coupling_pca → task2_predict
  → build_bundle，完成后前端即读到最新结果（随机种子固定，结果可复现）。

           task1_smoothing.py      任务1：滑动平均 + 指数加权平均趋势提取

六、输出 JSON 清单（output/，前端读取的数据基础）
--------------------------------------------------------------------------------
  region_stats.json        各海域 SST 均值/方差/极值/样本数（总览）
  correlation_matrix.json  13 维 Pearson 相关矩阵（维度互动强度热力图）
  pca_points.json          每条记录的三维主成分坐标 + region/co2/sst（3D 星云）
  pca_explained.json       三个主成分的方差解释率与累计值
  smoothed.json            各海域原始 SST + 滑动平均 + 指数加权（窗口 W 可调）
  prediction.json          双模型指标 + 测试集真实/预测 + 各海域误差 + 逐样本贡献分解
  feature_importance.json  随机森林各维度对 SST 预测的重要性（region one-hot 已聚合）
  forecast.json            随机森林对未来半年的 SST 预测 + 95% 置信区间
  records.json             清洗后记录明细（Query 查询 / Search 筛选 / 星云按维度着色）


七、算法要点
--------------------------------------------------------------------------------
  · 预处理：缺失值线性插值；按海域 3σ 剔除 SST 离群点；按海域+月份聚合统计。
  · 趋势提取：pandas rolling / ewm，不手写公式；窗口 W 前端滑块可实时调。
  · 降维耦合：StandardScaler 标准化后 PCA 降 3 维（去掉周期量 wind_dir）。
  · 多变量预测：特征 = 纬度/CO₂/风速/盐度/湿度/气压/洋流 + 月份 + 海域(one-hot)，
    标准化后 8:2 划分。主模型随机森林（出特征重要性），对比线性回归（出可解释
    系数，用于归因瀑布）。
  · 未来预测：未来特征 = 逐月气候态 + CO₂ 线性趋势外推；置信区间取森林内各决策树
    预测离散度 ±1.96σ（模型自带不确定性，非人为假设残差）。
